## 260716-7 转引自媒体被"据报道"洗白
- 规则：blog-researcher《转引自媒体的说法不算独立信源》/ blog-writer《转引自媒体的说法＝没有来源》
- 链条：自媒体"四海瞭望"称"因未满足法官要求，王女士最终输掉离婚案件"→ 新唐人电视台转引 → 研究文件标为"（来源：新唐人电视台转引报道）"→ 写手写成"据报道"进正文 → 用户删句。
- 出处：_pipeline_archive/research、review 下 260716-7 各版本。

## 事件号未记录 转发帖被当作来源
- 规则：blog-researcher《转发帖不作来源》/ blog-writer《转发帖不进文章》
- 链条：某案把博主 A 转发博主 B 的内容当来源，来源行只能写"A（转发 B 内容）"，读者无从对应原始出处。
- 出处：规则沉淀时未记事件号。

## 260707-2 / 260704-1 / 260703-2 tracker 来源归属三类错
- 规则：blog-researcher《Step 0》brief/来源不符即停下报回
- 链条：260707-2 来源 URL 挂到同批另一条无关帖子（湖北龙卷风微博）；260704-1 来源指向转发链末端需回溯原帖；260703-2 把江苏高邮"亡人事件"杀妻案与湖南母亲被造谣案缝进同一句 brief。
- 出处：2026-07-21 用户裁定，三次复现。

## 260708-4 / 260716-7 内部标签被当作文章标题
- 规则：blog-writer《标题自己写，不要照搬内部标签》
- 链条：内部标签"截瘫女子诉肇事男友案"（260716-7）被原样当标题发出，用户手改为"截瘫女子起诉肇事男友，维权期间遭网暴被迫终止治疗"；"脱口秀演员遭性骚扰反诉赔偿"（260708-4）被用户改为"脱口秀演员抵制偷拍者，反被起诉侵犯名誉权并被强制执行2.5万元赔偿"。
- 出处：2026-07-22 用户裁定，多次复现。

## 260716-7 / 260716-8 半匿名代称被二次化名
- 规则：blog-writer《半匿名代称已经是化名，原样沿用》
- 链条：2026-07-21 一天内复发两次——260716-7 把报道化名"白女士"改写成"林悦"、260716-8 把"高某某"改写成"李丽"，切断读者与原报道的对应关系，评审必退。
- 出处：2026-07-20 裁定、2026-07-21 重申。

## 事件号未逐一记录 同一案件被重复收录
- 规则：blog-researcher《Step 0 同案查重》
- 链条：重复事件多次走完研究、甚至走完写作后才被发现，整轮作废；同一跨国性侵案曾以四个不同标题被分别收录。
- 出处：审计 C9 动因；事件号未逐一记录。
